1. Basic Syntax

<!-- Element definition -->
<!ELEMENT element_name content>

<!-- Attribute definition -->
<!ATTLIST element_name
    attribute1 type requirement
    attribute2 type requirement>

<!-- Internal entity (constant) -->
<!ENTITY name "value">

<!-- Comment -->

2. Element Content Types

Type Meaning
(#PCDATA) Text content (e.g., "Hello", "123", "abc@def.com")
(child1, child2, child3) Child elements (exactly in this order)
EMPTY Empty element (e.g., <br/>)
ANY Anything (text, child elements, mixed)

3. Cardinality - How Many Times an Element Appears

Symbol Meaning Number of Occurrences
(none) Exactly once 1
? Optional 0 or 1
+ One or more 1, 2, 3, ... ‚àû
* Zero or more 0, 1, 2, 3, ... ‚àû

Examples:

Optional element
<!ELEMENT product (name, price, desc?)

desc is optional

At least 1
<!ELEMENT catalog (product+)

at least 1 product

0 or more
<!ELEMENT products (product*)

0 or more products

Multiple phones
<!ELEMENT person (name, surname, phone+)

at least 1 phone

4. Attribute Types

Type Meaning
CDATA Any text
ID UNIQUE Unique identifier in document
IDREF Reference to another element's ID
(a|b|c) Limited set of values (only a, b, or c)

5. Attribute Requirements

Requirement Meaning
#REQUIRED Required REQUIRED attribute
#IMPLIED Optional Optional attribute
"default" Default value
#FIXED "value" Fixed value (cannot be different)

6. Practical Examples

Text content (can be anything)

<!ELEMENT name (#PCDATA)>
<name>Logitech MX Mouse</name>

Required unique ID

<!ATTLIST product id ID #REQUIRED>
<product id="A001">...</product>

Limited set of values with default

<!ATTLIST price currency (EUR|USD|GBP) "EUR">
<price currency="USD">100</price>
<price>100</price>  <!-- defaults to EUR -->

Empty element with attribute

<!ELEMENT note EMPTY>
<!ATTLIST note text CDATA #IMPLIED>
<note text="Important!" />

Reference to another element

<!ATTLIST product supplierId IDREF #REQUIRED>
<!ATTLIST supplier id ID #REQUIRED>
<supplier id="S001">...</supplier>
<product supplierId="S001">...</product>

7. Quick Recipes

<!ELEMENT parent (child)>

8. Common Mistakes

ID cannot start with a number 
<product id="123">  <!-- ERROR! -->
CORRECT 
<product id="A123">  <!-- ‚úì CORRECT -->
Two elements with same ID 
<product id="A001">...</product>
<product id="A001">...</product>  <!-- ERROR! -->
Wrong order of elements 
DTD: <!ELEMENT product (name, price)>
<product><price>...</price><name>...</name></product>  <!-- ERROR! -->
EMPTY element with content 
DTD: <!ELEMENT note EMPTY>
<note>text</note>  <!-- ERROR! -->
CORRECT 
<note />  <!-- ‚úì CORRECT -->
Invalid value from limited set 
DTD: <!ATTLIST price currency (EUR|USD) "EUR">
<price currency="GBP">100</price>  <!-- ERROR! (GBP not allowed) -->

9. How to Link DTD with XML

External DTD file

<?xml version="1.0"?>
<!DOCTYPE root_element SYSTEM "vocabulary.dtd">
<root_element>
  ...
</root_element>

Internal DTD (inside XML document)

<?xml version="1.0"?>
<!DOCTYPE root_element [
  <!ELEMENT root_element (child*)>
  <!ELEMENT child (#PCDATA)>
]>
<root_element>
  ...
</root_element>
