1. What is XML Schema?

XML Schema (XSD) is a more powerful alternative to DTD for defining XML document structure:

  • Uses XML syntax (easier to learn and parse)
  • Supports data types (string, int, date, decimal, etc.)
  • More precise validation rules
  • Supports namespaces
  • Extensible and reusable

2. Creating XML with Schema Reference

In XML file - add schema reference

<?xml version="1.0" encoding="UTF-8"?>
<artikli xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="shema.xsd">
  <!-- your content here -->
</artikli>

Programmatically in C#

XmlElement rootArt = doc.CreateElement("artikli");
rootArt.SetAttribute("xmlns:xsi", "http://www.w3.org/2001/XMLSchema-instance");
rootArt.SetAttribute("noNamespaceSchemaLocation",
    "http://www.w3.org/2001/XMLSchema-instance", "shema.xsd");
docArt.AppendChild(rootArt);
Key attributes:
‚Ä¢ xmlns:xsi - declares the XML Schema Instance namespace
‚Ä¢ xsi:noNamespaceSchemaLocation - points to your XSD file

3. Basic XSD Schema Structure

4. Common XSD Data Types

<!-- Simple string element -->
<xs:element name="naziv" type="xs:string"/>

<!-- Integer element -->
<xs:element name="zaloga" type="xs:integer"/>

<!-- Decimal element -->
<xs:element name="cena" type="xs:decimal"/>

<!-- Date element -->
<xs:element name="zadnjaNabava" type="xs:date"/>

<!-- Restricted string (enumeration) -->
<xs:attribute name="kategorija">
  <xs:simpleType>
    <xs:restriction base="xs:string">
      <xs:enumeration value="elektronika"/>
      <xs:enumeration value="oblacila"/>
      <xs:enumeration value="obutev"/>
      <xs:enumeration value="hrana"/>
      <xs:enumeration value="drugo"/>
    </xs:restriction>
  </xs:simpleType>
</xs:attribute>
Common types:
‚Ä¢ xs:string - text
‚Ä¢ xs:integer - whole numbers
‚Ä¢ xs:decimal - decimal numbers
‚Ä¢ xs:date - dates (YYYY-MM-DD)
‚Ä¢ xs:boolean - true/false

5. Defining Elements with Attributes

<xs:element name="artikel">
  <xs:complexType>
    <xs:sequence>
      <!-- Child elements -->
      <xs:element name="naziv" type="xs:string"/>
      <xs:element name="cena" type="xs:decimal"/>
      <xs:element name="zaloga" type="xs:integer"/>
      <xs:element name="dobaviteljId" type="xs:string"/>
      <xs:element name="zadnjaNabava" type="xs:date"/>

      <!-- Empty element with attribute -->
      <xs:element name="razpolozljiv">
        <xs:complexType>
          <xs:attribute name="stanje" type="xs:string" use="required"/>
        </xs:complexType>
      </xs:element>
    </xs:sequence>

    <!-- Attributes on artikel element -->
    <xs:attribute name="id" type="xs:string" use="required"/>
    <xs:attribute name="kategorija" type="xs:string"/>
  </xs:complexType>
</xs:element>
Attribute usage:
‚Ä¢ use="required" - attribute is mandatory
‚Ä¢ use="optional" - attribute is optional
‚Ä¢ No use attribute = optional by default

6. Occurrence Constraints

<!-- Element must appear exactly once (default) -->
<xs:element name="naziv" type="xs:string"/>

<!-- Element is optional (0 or 1) -->
<xs:element name="opis" type="xs:string" minOccurs="0"/>

<!-- Element can appear 0 or more times -->
<xs:element name="artikel" minOccurs="0" maxOccurs="unbounded"/>

<!-- Element must appear at least once -->
<xs:element name="artikel" minOccurs="1" maxOccurs="unbounded"/>

<!-- Element appears between 1 and 5 times -->
<xs:element name="telefon" minOccurs="1" maxOccurs="5"/>
Quick reference:
‚Ä¢ minOccurs="0" maxOccurs="1" = optional (0 or 1)
‚Ä¢ minOccurs="0" maxOccurs="unbounded" = 0 or more
‚Ä¢ minOccurs="1" maxOccurs="unbounded" = 1 or more
‚Ä¢ No occurrence attributes = exactly once

7. XML Schema Validation in C#

Setup validation

using System.Xml;
using System.Xml.Schema;

static bool validacijaOK = true;

static void ValidirajXML(string pot)
{
    validacijaOK = true;

    try
    {
        XmlReaderSettings settings = new XmlReaderSettings();
        settings.ValidationType = ValidationType.Schema;
        settings.ValidationEventHandler += new ValidationEventHandler(ValidationCallBack);

        // Add schema file
        settings.Schemas.Add(null, "shema.xsd");

        XmlReader reader = XmlReader.Create(pot, settings);

        while (reader.Read()) ;  // Read entire document to validate
        reader.Close();

        if (validacijaOK)
        {
            Console.WriteLine("‚úÖ XML is valid!");
        }
    }
    catch (Exception ex)
    {
        Console.WriteLine($"‚ùå Validation error: {ex.Message}");
        validacijaOK = false;
    }
}

// Callback for validation errors
private static void ValidationCallBack(object sender, ValidationEventArgs e)
{
    Console.WriteLine($"‚ùå Validation error: {e.Message}");
    validacijaOK = false;
}

8. Safe Validation Pattern with Temp Files

static void DodajArtikel()
{
    // ... get user input ...

    try
    {
        XmlDocument doc = new XmlDocument();
        doc.Load("artikli.xml");

        // ... create and add new element ...

        // Save to temporary file first
        string tempPath = "temp_artikli.xml";
        doc.Save(tempPath);

        // Validate temp file
        ValidirajXML(tempPath);

        if (validacijaOK)
        {
            // Only replace original if validation passed
            File.Copy(tempPath, "artikli.xml", true);
            File.Delete(tempPath);
            Console.WriteLine("\n‚úÖ Article successfully added and validated!");
        }
        else
        {
            // Delete temp file if validation failed
            File.Delete(tempPath);
            Console.WriteLine("\n‚ùå Article NOT added due to validation error!");
        }
    }
    catch (Exception ex)
    {
        Console.WriteLine($"\nError: {ex.Message}");
    }
}
Always validate before saving! Use the temp file pattern to avoid corrupting your XML files.

                        <xs:enumeration value="da"/>
                        <xs:enumeration value="ne"/>
                      </xs:restriction>
                    </xs:simpleType>
                  </xs:attribute>
                </xs:complexType>
              </xs:element>
            </xs:sequence>

            <xs:attribute name="id" type="xs:string" use="required"/>
            <xs:attribute name="kategorija" use="required">

                  <xs:enumeration value="obutev"/>
                  <xs:enumeration value="hrana"/>
                  <xs:enumeration value="drugo"/>
                </xs:restriction>
              </xs:simpleType>
            </xs:attribute>
          </xs:complexType>
        </xs:element>
      </xs:sequence>
    </xs:complexType>
